# Security Policy

## 🛡️ Responsible Use

ShadowScan is a security testing tool designed for **authorized use only**. Users must ensure they have proper authorization before scanning any networks or systems.

### Authorized Use Cases

- ✅ Scanning your own systems and networks
- ✅ Authorized penetration testing engagements
- ✅ Security research with proper permissions
- ✅ Educational purposes in controlled environments
- ✅ Bug bounty programs within scope

### Unauthorized Use Cases

- ❌ Scanning systems without permission
- ❌ Any illegal activities
- ❌ Malicious hacking attempts
- ❌ Disruption of services
- ❌ Data theft or exfiltration

---

## 🔒 Security Considerations

### For Users

1. **Always obtain written authorization** before scanning
2. **Use appropriate scanning rates** to avoid network congestion
3. **Document all scanning activities** for compliance
4. **Follow responsible disclosure** if vulnerabilities are found
5. **Comply with local laws** and regulations

### For Developers

1. **Never add malicious functionality** to the codebase
2. **Report security issues** responsibly
3. **Keep dependencies updated** to prevent vulnerabilities
4. **Follow secure coding practices**

---

## 📣 Reporting Security Issues

If you discover a security vulnerability in ShadowScan itself (not vulnerabilities found using ShadowScan), please report it responsibly.

### How to Report

1. **DO NOT** open a public issue
2. Email the author directly: security concerns should be reported privately
3. Include detailed information:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### Response Timeline

- Initial response: Within 48 hours
- Assessment: Within 7 days
- Fix development: Depends on severity
- Public disclosure: After fix is released
